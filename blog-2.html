Software as Service (SaaS)

  • Author
  • Nov 15, 2017
  • 1
  • 3

SaaS is software offered to clients as a service where clients do not need to update the software offered but when the provider update software then the client follow the update. Or in other words, SaaS is a software with delivery method that serves access to the software and simultaneously its functions as a web-based service. For access to business interests SaaS can provide services at a cheaper price than software with licensed applications because SaaS is paid on a regular basis as needed (daily, monthly or yearly). Characters from SaaS include commercial software for web access, centralized location and managed by software, software with "one to many" model, users do not need to handle software and patch upgrades, and APIsallows integration between different parts of the software. Of course, these characters become the advantages and disadvantages for SaaS providers and users tailored to the needs. Some of the advantages possessed by SaaS are as follows:

  1. Easy to use because SaaS applications do not require more than one web browser.
  2. Cheap, SaaS is affordable for small businesses and individuals to make payments because payouts are made with various models as needed.
  3. Scalability means to meet customer demand, SaaS applications can be easily upgraded or downgraded without having customers do infrastructure computing.
  4. When data is stored in cloud then application is safe against data loss.
  5. Customers do not need to install / uninstall binary code on the machine and can run on various software due to the nature of SaaS delivery over the internet.
  6. For the benefit of collaboration in a team in performing a project then SaaS is better because the location of data is centralized and stored on the internet.

In software analysis for business purposes, software cost allocation is based on three things: software that becomes the program and the actual data from customers to perform computing and information processing, hardware that serves customers to access the software, and service / service of the software provided. In desktop applications, the proportion of costs is spent on hardware and professional services where hardware is purchased so that the software can be run in accordance with minimum computer skills requirements, while professional service is dominated because it involves human resources to utilize the software offered. In SaaS applications, the proportion of software is more dominant because users can access software easily via the web, while the proportion of hardware and professional services is smaller because more flexible software can be used for various computer capabilities and human resources can be minimized with the internet access.

Implementation of SaaS in mineboard applications is done to facilitate in updating and evaluation of mining reporting and simplify the calculation of expenses incurred when the company will use this application. This is because every company has a difference in doing mining activities such as activity time, target poduksi within a certain period. So the SaaS system is appropriately applied to optimize the needs of the mining company. The intended optimization is paid software services based on time, computer capabilities, and company needs in accordance with the amount of data generated from the reporting of mining activities.
